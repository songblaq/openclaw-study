# 스킬 시스템

## 스킬이란?
스킬 폴더는 에이전트에게 도구 사용법을 가르친다. 각 스킬은 YAML 프론트매터 + 지침이 포함된 `SKILL.md`를 갖는다.

## 위치 (우선순위)
1. `<workspace>/skills` (최상위)
2. `~/.openclaw/skills` (관리/로컬)
3. 번들 스킬 (최하위)

같은 이름의 스킬: 작업공간이 우선.

## 스킬 형식

```markdown
---
name: nano-banana-pro
description: Gemini 3 Pro로 이미지 생성
metadata: { "openclaw": { "requires": { "bins": ["uv"], "env": ["GEMINI_API_KEY"] }, "primaryEnv": "GEMINI_API_KEY" } }
---

# 여기에 지침...
```

## 게이팅 (로드 시간 필터)

스킬은 다음 기준으로 필터링:
- `requires.bins` - PATH의 바이너리
- `requires.env` - 환경 변수
- `requires.config` - 참이어야 하는 설정 경로
- `os` - 플랫폼 제한 (`darwin`, `linux`, `win32`)

## 설정 오버라이드

```json5
{
  skills: {
    entries: {
      "nano-banana-pro": {
        enabled: true,
        apiKey: "GEMINI_KEY_HERE",
        env: { GEMINI_API_KEY: "KEY" }
      },
      peekaboo: { enabled: true },
      sag: { enabled: false }
    }
  }
}
```

## ClawHub (스킬 레지스트리)

브라우저: https://clawhub.com

```bash
# 스킬 설치
clawhub install <skill-slug>

# 전체 업데이트
clawhub update --all

# 동기화 (스캔 + 게시)
clawhub sync --all
```

## 멀티 에이전트 스킬
- **에이전트별**: `<workspace>/skills` (에이전트 전용)
- **공유**: `~/.openclaw/skills` (모든 에이전트)
- 추가 폴더: 설정의 `skills.load.extraDirs`

## 토큰 영향
스킬 로드 시 시스템 프롬프트에 주입:
- 기본 오버헤드: ~195자
- 스킬당: ~97자 + 이름 + 설명 + 위치

## 보안 참고
- 서드파티 스킬은 **신뢰하지 않는 코드**로 취급
- 활성화 전에 스킬 읽기
- 위험한 도구는 샌드박스 실행 권장
- `env`와 `apiKey`는 호스트 프로세스에 주입 (샌드박스 아님)

## 스킬 워처
`SKILL.md` 변경 시 자동 새로고침:

```json5
{
  skills: {
    load: {
      watch: true,
      watchDebounceMs: 250
    }
  }
}
```

## 세션 스냅샷
세션 시작 시 스킬 스냅샷. 변경 사항은 다음 새 세션에 적용 (또는 워처로 핫 리로드).
