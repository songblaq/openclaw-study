# 설정 가이드

## 핵심 원칙
커스터마이징은 **저장소 외부**에 보관:
- 설정: `~/.openclaw/openclaw.json`
- 작업공간: `~/.openclaw/workspace`

## 부트스트랩

```bash
openclaw setup
```

## 두 가지 워크플로우

### 1. 안정적인 워크플로우 (macOS 앱)
1. OpenClaw.app 설치 및 실행
2. 온보딩 권한 완료
3. 로컬 모드로 게이트웨이 실행 (앱이 관리)
4. 채널 연결: `openclaw channels login`
5. 확인: `openclaw health`

### 2. 최신 기능 (터미널 게이트웨이)
TypeScript 개발용 핫 리로드:

```bash
pnpm install
pnpm gateway:watch
```

macOS 앱을 로컬 모드로 설정 → 실행 중인 게이트웨이에 연결.

## 자격증명 저장 위치

| 항목 | 위치 |
|------|------|
| WhatsApp | `~/.openclaw/credentials/whatsapp/<accountId>/creds.json` |
| Telegram 봇 토큰 | config/env 또는 `channels.telegram.tokenFile` |
| Discord 봇 토큰 | config/env |
| Slack 토큰 | config/env |
| 페어링 허용 목록 | `~/.openclaw/credentials/<channel>-allowFrom.json` |
| 모델 인증 프로필 | `~/.openclaw/agents/<agentId>/agent/auth-profiles.json` |
| 레거시 OAuth | `~/.openclaw/credentials/oauth.json` |

## Linux 참고
Linux는 systemd **사용자** 서비스 사용. 로그아웃 시 게이트웨이 중지 방지:

```bash
sudo loginctl enable-linger $USER
```

## 업데이트
- `~/.openclaw/`는 "내 것"으로 유지
- 소스 업데이트: `git pull` + `pnpm install`
- 개인 프롬프트/설정은 저장소 외부에 보관
